Developer documentation for Oratorio

How can I find the source code?

Click on releases to download the latest release version of oratorio. Or clone the repository to obtain the latest development version of oratorio.

How to build the software

Install Python 3.5 and Django 1.10.

How to test the software

Run the testsuite by running python manage.py test.
Run a development server by running python manage.py runserver.

The layout of the directory structure

oratorio/ - Project directory
 manage.py - Django's admin utility file
 coach/ - Main application directory
  tests/ - Tests for main application
  settings.py - Settings for the main application
  models.py
  views.py
  urls.py
 templates/ - HTML templates for the whole project
 static/ - Images, css and other static files for the whole project

How to set up an automated daily (or more frequent) build and test

Set up Travis CI by following the getting started guide.

How to release a new version of your software

To release a new version, package the code in a tar file and create a new release through Github's Releases feature. Detailed instructions here.
Our project is open source so there will not be any private releases. If a developer only release is needed, a commit can be tagged to signify that release.

How to access the list of outstanding bugs and the list of resolved bugs, and how to resolve a bug

All bugs and issues should be reported here.
Please follow the issue template and be as informative as possible.
